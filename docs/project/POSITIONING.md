## 0. 一句话定位

**中文：** OxyGenie 是**能私有化部署的团队版 Cowork**——把"让 AI 真正动手干活"的体验，
做成团队可自部署、数据不外流、还能跑性价比模型的 Web 工作台。

**English:** **Kin is the self-hostable, team-grade alternative to Claude Cowork** —
an autonomous agent workspace your whole team runs on its own infrastructure, with
multi-model support and your documents/conversations/audit logs staying on your servers
(model calls go only to the endpoint you choose; provider-neutral, not air-gapped — see §4.5).

> 记忆锚点：**"Cowork，但能私有化、给整个团队用。"**

## 1. 为什么用 "Cowork alternative" 这个框

Cowork 是 Anthropic 给非开发者做的桌面 Agent——它的杀手级体验不是"聊天"，而是
**"AI 真的去动你的文件、跑代码、完成任务"**。这正是 OxyGenie 想给团队的体验。

借这个框的三个好处：

1. **一句话说清品类。** 不用解释"Agent 平台"这种抽象词，对标一个具体产品，认知成本最低。
2. **天然带出差异化。** Cowork = 单机 / 单人 / 云端 / 绑定 Anthropic。OxyGenie 的每一个
   "反面"都是卖点：**自部署、多人团队、私有化、多模型**。
3. **蹭到 Cowork 教育市场的红利。** Anthropic 在帮我们教育"Agent 能替你干活"这件事，
   我们接住那批"想要但不能用云端/不能单机/想给团队用"的需求。

⚠️ **风险（见 §7）：** Cowork 目前是 research preview，国内认知度低。所以**海外叙事直接用
"Cowork alternative"，国内叙事要换一个更易懂的锚点**（见 §6）。

## 3. 目标用户（ICP）—— 收敛于 2026-06-13

**画像不是行业，是结构：数据极度敏感、人数 <30、没有专职 IT、愿为隐私付费的小团队。**
典型：律所、基金、家族办公室、科研/高校实验室、精品工作室，也包括大公司里的敏感小团队。
共同结构：

- 数据碰不得公有云 AI（法律卷宗 / 研究 IP / 持仓与尽调 / 家族资产）；
- 人少但要团队共享知识与产出——**桌面单机 Cowork 不够**；
- 没有运维——**Dify/k8s 太重**；
- 不差钱 / 愿为"绝对私有"付费。

**物理形态：一台 Mac mini 装下，插电即用。** 一句话品类 = **"数据敏感小团队的 AI 私有一体机"**
（像 Synology NAS，但装的是 AI agent）。*因为只用 API、不跑本地推理（见 §4.5），盒子只承载 App+编排+RAG+
向量库+Web，**Mac mini 算力绰绰有余**，不再需要 Mac Studio。`docs/deployment/mac-mini.md` + Cloudflare
Tunnel 部署路径已验证——GTM 形态是把已有部署能力包装成产品叙事，不是新工作量。*

**滩头阵地（beachhead）= 律所先打，基金次之，家办留后**（依据见
`research/2026-06-beachhead-and-positioning-research.md`，三类打分 律 25 / 基 21 / 家 17）。
- **律所（精品/中小所）= 第一滩头**：基数最厚、"不能上云"痛点既硬又普适（**有联邦判例：用公开 AI 工具
  会刺破 attorney-client privilege**，几乎为私有部署量身定制）、工作流最同质、最易触达且愿公开背书。
- **基金（小型/精品）= 第二滩头/上探**：ARPU 最高、价格空档最大（被 Hebbia/AlphaSense 的 $100K+ 挡门外），
  但渠道封闭、案例忌讳公开、工作流容错低。
- **家办 = 后期高端定制**：痛点+钱都在，但稀缺、极封闭、一户一策、拿不到署名案例。
代价（要正视）：律所赛道已是红海（Harvey/CoCounsel/Spellbook/Clio），保守、信任门槛高、销售周期长——
**早期靠顾问式高触达销售 + 标杆案例，开源负责攒可信度而非直接获客**。
⚠️ **三类里先把律所做深一个子类**（有暖线索者优先），别同时铺三类。

### 两层 ICP：采用引擎 vs 营收引擎（2026-06-14 补）

同一个产品、同一结构画像，但分两层人群、两种打法，**互相喂养**：

| | 🌱 采用 / 社区引擎（bottom-up） | 💰 营收引擎（top-down） |
|---|---|---|
| **人群** | **高校/研究生科研团队**、精品工作室、初创小团队 | 律所 → 基金 → 家办 |
| **特点** | **用 AI 多、钱少**、技术好能自部署、数据敏感（未发表研究/IP、课题数据、IRB/经费约束不能进公网 AI） | 付费能力强、合规刚需、信任门槛高 |
| **打法** | 免费开源、自助上手、**provider 中立按最便宜模型路由控成本**（GLM/DeepSeek/免费额度） | 顾问式高触达、垂直技能模板、标杆案例、企业版付费 |
| **负责** | star / 反馈 / 打磨 PMF / 沉淀技能 / 口碑背书 | 收入 |

**为什么这样分：** 科研/学生团队是天然的开源采用者和布道者，用量大、迭代快，**负责把产品打磨到 PMF、
攒社区可信度**；这份可信度再反哺律所/基金的顾问式销售。这是 LibreChat / Dify 走通的 open-core 路径。
⚠️ **风险：** 两层人群消息要分开讲，别让"给学生的省钱叙事"污染"给律所的数据主权叙事"，否则产品定位会糊。

**扩张路径：** 科研/工作室（采用、攒口碑）↗ + 律所标杆（营收）→ 基金 → 家办 → 开发者自部署生态。

> **地域纠结被这个 ICP 化解了：** 这个画像在国内外一模一样地存在、痛点也一样。
> 楔子是"组织类型 × 规模"，**与地域无关**——同一把楔子两个市场都能扎（呼应 §6）。
> 科研/高校这一层在中国尤其量大（呼应 §6 国内主场）。

## 4. 差异化支柱（对外只讲这 4 条）

1. **私有化 / 数据主权** —— 自部署，**资料库与系统记录（文档/知识库/会话/审计/权限）留在自己服务器**；
   模型调用走你选定的零留存合规端点，且**不锁定任何一家**（口径见 §4.5，别吹成 air-gap）。
   这是对 Cowork 和一切"语料也托管在厂商云"的产品最硬的差异。
2. **团队，而不是单机单人** —— 多用户、会话/知识库共享、按用户隔离与权限。Cowork 是给个人的，
   我们是给一个组织的。
3. **真的能干活（Agent 能力，不只是聊天）** —— Skills Store + MCP 一键扩展、Artifacts 生成、
   Python 沙盒执行、实时预览分享。对标 Cowork 的"动手"体验，碾压普通聊天产品。
4. **多模型 / 成本可控** —— 经 ARK（火山）网关跑 GLM/豆包等性价比模型，也能用 Claude。
   团队规模化用 AI 时，成本是真实决策因素。

> 这 4 条的顺序就是优先级：**安全 → 团队 → 能干活 → 省钱**。
> （"省钱"对实验室/工作室成立；对律所/基金/家办，第 4 条换成"用最强模型也不出盒子"——见 §4.5。）

## 4.5 模型策略：只用 API、不做本地模型、不锁提供商（2026-06-14 决定，决定隐私叙事口径）

**决定：OxyGenie 不内置、不依赖本地模型，统一走模型 API，且 provider-agnostic（不锁定任何一家）。**
这条**重写了隐私叙事**，必须讲准，不能过度承诺：

- ❌ **不能再说**"air-gapped / 数据一步不出门 / 完全离线 / 纯本地"。模型调用那段内容**确实会发给你选的 API**。
- ✅ **能说、且依然有差异化的是**：**"你的文档、知识库、会话、审计、权限、工作流，全在你自托管的盒子里
  （不在厂商 SaaS 上）；模型调用只把当次推理内容，发给你自己选定的、零留存/不训练的合规端点。"**

**为什么这条路仍然成立、甚至更优：**

1. **质量命门被解决**：直接用 Claude/GPT 顶配 → 律所"零容忍幻觉"那条魔鬼代言人（§3）被绕过。
2. **硬件叙事反而更稳**：盒子只跑 **App + 编排 + RAG + 向量库 + Web**，**不跑推理**——**一台 Mac mini 绰绰有余**，
   research 报告里"Mac mini 跑不动大模型 / 要上 Mac Studio"的问题**直接消失**。
3. **provider 中立 = 真卖点**：不被任何模型商绑定（对位 Harvey 锁死自家栈），按合规/成本/质量自由切换——
   今天 Claude、明天 GPT、要便宜上 GLM/DeepSeek。这本身就是对律所/基金有说服力的一句话。
4. **极致私有仍可达**：要 air-gap 的客户，把 API 指向**他自己 VPC 里的推理端点**（Bedrock 自有账号 / 私有部署）
   即可——**我们不挑端点，只是不内置本地模型**。私有化的"最后一公里"交给客户的端点选择。

**对律所/基金的合规底线：** 默认只接**零数据保留 + 不用于训练 + 可签 DPA/BAA** 的端点，
并在产品里**明确告诉用户"这段内容会发到哪个端点"**（可见的数据边界）。

> **差异化一句话（修订）：** 不再是"纯本地、什么都不出门"，而是
> **"你的资料库和系统记录留在你的盒子里；模型用谁、发去哪，由你定，还不绑定任何一家。"**
> **卖的是「数据主权 + 不锁定」，不是 air-gap。** 对外文案务必守住这个口径，别让销售吹成"绝对不外流"。

## 7. 待解决的张力与风险（必须正视）

你的四个选择内部有三处需要你拍板的张力，按优先级排：

1. **【最需决策】"两者都要"≈ 没有主战场——而 OpenWork 的存在让这个决策更紧迫。**
   开源项目早期最怕分散，而**海外"开源版 Cowork"这条赛道已被 OpenWork（14.6k★）占住**，
   正面拼 star 是逆风局。建议：**用"中国为主战场、海外做差异化声量"的非对称打法**——
   产品采用和早期付费押在**国内**（数据安全 + 性价比模型 + 合规是国内团队真实刚需，
   而这正是 OpenWork 的空档）；GitHub 英文 README 不和 OpenWork 抢同一句话，而是打
   *team-native / web / Claude Agent SDK* 的差异化标签，攒信任反哺国内可信度。
   *需要你确认：是接受"国内为主、海外差异化引流"的主次，还是真要 50/50 正面刚 OpenWork？*

2. **"开源为主" × "买家是团队负责人" 的衔接。**
   开源不直接产生收入，团队负责人要的是"能用、有人兜底"。**必须想清 open-core 的免费/付费线**：
   - 免费（社区版）：完整自部署、Skills/MCP、单组织多用户。
   - 付费（企业版/托管）：SSO、审计日志、SLA、托管运维、优先支持、团队治理后台。
   *现在 `usage_record` / `audit_log` / `/admin` 已经有了底子，离企业版不远。*

3. **"Cowork alternative" 的认知度风险。**
   Cowork 太新，国内几乎没人知道。所以**这个锚点只在海外用**，国内换成"通用 Agent + 私有化"
   （见 §6）。同时要押注 Cowork 会火——如果它没起来，这个对标的杠杆就弱了，需要 plan B 锚点
   （备选："私有化的 Manus" / "团队自部署的 AI 员工"）。
