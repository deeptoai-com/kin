# 导航信息架构（IA）重构 PRD

**日期**：2026-06-12
**状态**：已与 Owner 讨论定稿，分阶段实施中
**关联**：[[2026-06-projects-collaboration-prd]]（Projects = 团队协作容器 Model A）、单入口路由（PR #152）

---

## 1. 背景与问题

单入口改造（#152）后，聊天与项目收敛成 ChatGPT 式单入口，但导航的信息架构出现职责错位：

- **主侧边栏**（`AppSidebar`，shadcn sidebar）装满了 agent 功能模块导航：智能体 / 项目 / 能力中心 / 文档 / 仪表盘 + 首页链接。
- **副侧边栏**（`ProjectsRail`，固定 `w-72`）只做了"新建聊天 + 项目列表平铺 + 最近"。

结果：① "项目"同时出现在主、副两处，用户分不清；② 主侧边栏实际承担了副侧边栏该干的事（agent 工作台导航）；③ 副侧边栏把"项目列表"平铺，没有 ChatGPT/Claude Desktop 那种"项目作为入口条目 + 最近在下"的清晰分层。对非技术用户（律师/财务/白领/学生，见 [[oxygenie-target-users]]）尤其困惑。

## 2. 目标 IA：两级导航

清晰的两级：**主侧边栏选模块 → 副侧边栏在「智能体」模块内导航**。

### 2.1 主侧边栏 = 独立功能模块（L1）

三个互相独立、不一定依赖 agent 的功能模块：

| 模块 | 路由 | 说明 |
|---|---|---|
| **智能体** | `/agents/c` | agent 工作台入口（进入后副侧边栏出现） |
| **能力中心** | `/agents/capabilities` | skills + MCP 策展（Capability Center） |
| **知识库** | `/agents/documents` | 文档/知识库。**独立功能，不依赖 agent**，故单列主侧边栏 |

**Owner 决策（2026-06-12）**：
- **不引导去首页**：首页只是 marketing homepage，logo 不再 `Link to "/"`。
- **不做"团队层/共享项目/管理员预设"**：目前没有"共享项目"这类团队级功能，主侧边栏暂不承担团队层职责（曾讨论过"管理员预设快捷方式"方案，因功能尚不存在而搁置）。
- **仪表盘（charts）移出**：不属于这三个模块，归到管理面板（admin）。
- **管理面板**：管理员专属，独立于三模块，继续仅对 `systemRole=admin` 显示。
- 以后是否新增模块，随能力扩展再定。

### 2.2 副侧边栏 = 智能体工作台（L2）

仅在「智能体」模块下出现。最终条目：

```
[+ 新建任务]          ← 开始入口（动作按钮）
🔍 搜索               ← ⌘K 全局对话搜索（新增）
📁 项目               ← 入口条目，点进去是 Projects 落地页（不再平铺列表）
✨ 作品               ← Live artifacts 聚合页（待建，阶段 3）
──────
置顶                  ← Pinned（新增）
最近                  ← Recents（散聊，不含项目内会话）
```

- 切换条目时**保留当前聊天状态**（run loop / session / store 已是 module-level，跨路由不丢——单入口改造时即如此）。

### 2.3 Projects 落地页

点副侧边栏「项目」进入。空状态有引导文案 + 新建按钮；有项目时是**卡片网格**：项目名 + 成员头像 + 文档/会话数 + 更新时间 + 搜索 + 排序。

**关键差异（不可照搬 Claude Desktop）**：Claude Desktop 的 Project 关联本地文件夹路径（桌面单机应用）；oxygenie 的 Project 是**服务端团队协作容器**（成员、共享、`branched_from`，Model A）。故卡片显示**团队属性**（成员/文档数/共享状态），**不是**本地路径。

## 3. 缩放（折叠）规则

当前是**反的**：主侧边栏可折叠成图标条（`collapsible="icon"`，默认折叠），副侧边栏 `w-72` 写死不可折。用户写长文/看预览时最该收起的副侧边栏反而收不起来。

**目标规则**：职责分离，不给两个对等折叠键。
- **主侧边栏（模块条）= 窄图标条常驻**（icon-only ~64px，3 模块 + 设置 + 头像，不值得折叠，常驻让用户随时知道在哪个模块）。
- **副侧边栏（工作台）= 一个折叠键**（顶部），收起后内容区变宽。这是唯一需要折叠的那条。
- 极致专注场景可选一个"全隐藏"总开关把两条都收掉（次要，默认不做）。
- 即：把现有那个折叠图标的作用从"折主条"改成"折副条"。

## 4. 搜索

当前**没有对话搜索**（`command.tsx` cmdk 组件在仓里，但只接了技能搜索）。几百条对话只能肉眼翻"最近"。

**方案**：副侧边栏固定"搜索"入口 + ⌘K 全局唤起 command palette：列近期对话 + **项目标签** + ⌘1-9 快速跳。
- **先做**：按标题 + 项目搜（DB 直查，轻量）。
- **后续增强**：全文搜索（搜进消息内容）走 Meili。
- cmdk 组件现成，复用。

## 5. 分阶段实施

| 阶段 | 范围 | 性质 |
|---|---|---|
| **1. 主侧边栏精简** | `AppSidebar` 改为三模块（智能体/能力中心/知识库）；移除项目、仪表盘；logo 去首页链接；admin 保留 | 纯改一个组件，零新功能、低风险 |
| **2. 副侧边栏重组** | 项目改入口条目 + 加置顶；`w-72` 改可折叠；加搜索条目 + ⌘K palette（先标题/项目）；Projects 落地页卡片网格 | 重组 + 重做页面 + 搜索/折叠 |
| **3. 作品聚合页** | Live artifacts（preview/artifact 数据汇成页）；全文搜索增强 | 新功能，单独排期 |

阶段 1+2 把现有功能重新归位 + 重做 Projects 页 + 补搜索/折叠，IA 即立到位；阶段 3 是新功能往后放。

## 6. 与产品定位的契合

oxygenie 北极星 = 面向中小团队的私有化部署、单组织多用户（见 `VISION.md`）。本 IA：
- 主侧边栏「智能体/能力中心/知识库」= 当前真实能力，不硬造尚不存在的团队层。
- 副侧边栏 = 个人 agent 工作台（对齐 Claude Desktop 心智）。
- Projects = 团队协作容器（区别于 Claude Desktop 个人版的本地文件夹）。
- 团队层（共享项目/管理员预设）作为未来能力扩展的预留方向，不在本次范围。

## 7. 项目管理（Owner 2026-06-12 截图细化 — 阶段3，慢慢实现）

**项目分两类，菜单按角色不同**：
- **自己创建（owner）**：分享项目 / 重命名项目 / 删除项目
- **加入团队他人的（member）**：分享项目 / 离开项目；详情页 member 还有「项目设置」「举报项目」

**两个管理入口**：
1. **副侧边栏**：项目行 hover → ··· 菜单 + ✏️ edit 图标（快捷重命名）。
2. **项目详情页**：顶部成员头像 + 分享按钮 + ··· 菜单（owner: 分享/重命名/删除；member: 项目设置/举报项目/离开项目）。

**项目设置弹窗**（详情页 ··· →「项目设置」）：
- **项目名称**：带 emoji/图标选择器
- **指令**：自定义 AI 回复方式（项目级 system 指令；占位文案"用西班牙语回答…回答简短突出重点"）
- **记忆**：`默认`（可访问外部聊天记忆，反之亦然）/ `仅限项目`（只访问自身记忆，对外不可见）——**此设置创建后不可改**
- **owner** 弹窗底部有「删除项目」按钮

**已做（阶段2c-2b）**：副侧边栏项目 ··· 基础菜单（owner 分享/重命名/删除、member 分享/离开），重命名/分享 dialog。
**待做（阶段3，慢慢实现）**：① 副侧边栏 ✏️ edit 快捷入口 ② 项目详情页顶部管理区（成员头像/分享/··· 菜单）③ 项目设置弹窗（emoji 选择器 + 指令 + 记忆 + 删除）④ 举报项目 ⑤ member 设置只读视图。其中「指令」「记忆」是项目能力的核心（影响 RAG/记忆隔离），需配合后端字段（`projects.instructions` 已有；记忆 scope 待设计）。
