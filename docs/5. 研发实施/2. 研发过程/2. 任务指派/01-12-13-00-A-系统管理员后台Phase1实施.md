## 完成内容

### 1. 数据库 Schema 变更 ✅
**文件**：`src/db/schema/auth.schema.ts`
- 添加 `systemRole` 字段到 `user` 表（'admin' | 'user'）
- 默认值为 'user'
- 添加数据库迁移脚本：`migrations/add_system_role/001_add_system_role.sql`

### 2. 注册逻辑修改 ✅
**文件**：`src/server/auth.server.ts`
- 添加 `onCreate` 钩子
- 第一个注册用户自动设置为 `admin`
- 后续用户默认为 `user`

### 3. 系统管理员服务端函数 ✅
**文件**：`src/server/admin.server.ts`

**函数列表**：
- `requireSystemAdmin()` - 验证系统管理员身份
- `getAllUsers()` - 获取所有用户及其 Credits
- `addUserCredits()` - 手动给用户充值 Credits
- `updateUserSystemRole()` - 更新用户系统角色
- `getAllOrganizations()` - 获取所有组织
- `createOrganizationAsAdmin()` - 创建组织（指定 Owner）
- `deleteOrganization()` - 删除组织
- `getOrganizationDetails()` - 获取组织详情和成员列表

### 4. Admin 后台布局 ✅
**文件**：
- `src/components/admin/admin-nav.ts` - 导航配置
- `src/components/admin/AdminLayout.tsx` - 布局组件
- `src/routes/admin/route.tsx` - 主路由
- `src/routes/admin/index.tsx` - Dashboard 首页

**布局特性**：
- 左侧 Sidebar 导航
- 响应式设计
- 深色模式支持
- 权限验证（非管理员重定向）

### 5. 用户管理页面 ✅
**文件**：`src/routes/admin/users/route.tsx`

**功能**：
- ✅ 用户列表（邮箱、角色、Credits 余额、订阅状态）
- ✅ 手动充值 Credits（选择类型：购买/赠送/补偿）
- ✅ 更新系统角色（提升/降级为管理员）
- ✅ 统计卡片（总用户数、管理员数、总 Credits）

### 6. 组织管理页面 ✅
**文件**：`src/routes/admin/organizations/route.tsx`

**功能**：
- ✅ 组织列表（名称、Owner、成员数、权限配置）
- ✅ 创建组织（指定 Owner、配置权限）
- ✅ 删除组织（二次确认）
- ✅ 查看组织详情（成员列表）
- ✅ 统计卡片（组织总数、成员总数）

## 使用指南

### 步骤 1：运行数据库迁移

```bash
# 连接到数据库并执行迁移
psql postgresql://user:password@localhost:5432/ex0 < migrations/add_system_role/001_add_system_role.sql
```

### 步骤 2：第一个用户自动成为管理员

1. 启动应用
2. 注册第一个用户（gaopeng@winko.ai）
3. 自动成为系统管理员
4. 访问 `/admin` 查看后台

### 步骤 3：使用管理员后台

#### 访问地址
```
http://localhost:5000/admin
```

#### Dashboard
- 系统概览
- 快速操作入口

#### 用户管理 (`/admin/users`)
- 查看所有用户
- 点击 "Add Credits" 充值
- 点击盾牌图标修改角色

#### 组织管理 (`/admin/organizations`)
- 点击 "Create Organization" 创建
- 指定组织 Owner（从现有用户选择）
- 配置权限模式和 Bash 工具
- 查看组织成员

## 数据流图

```
用户注册
  ↓
auth.server.ts onCreate 钩子
  ↓
检查是否是第一个用户
  ↓
是 → system_role = 'admin'
否 → system_role = 'user'
```

```
访问 /admin
  ↓
requireSystemAdmin() 验证
  ↓
system_role === 'admin'?
  ↓
是 → 显示后台
否 → 重定向到 /agents/claude-chat
```

2. **Credits 充值**：没有撤销功能（需要手动添加负数冲正）
3. **统计图表**：Dashboard 暂时显示占位符，未实现真实数据
